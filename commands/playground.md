# Playground

単一HTMLファイルで完結するインタラクティブなツール・デモ・ビジュアライゼーションを生成する。
外部依存ゼロ。ブラウザで開くだけで動く。

## タスク

$ARGUMENTS

## 制約

### 技術的制約

- **単一ファイル**: HTML + CSS + JavaScript を1つの `.html` ファイルに収める
- **外部依存なし**: CDN リンク、外部CSS、外部JS は一切使わない
- **インライン完結**: `<style>` タグ内に CSS、`<script>` タグ内に JavaScript
- **モダンブラウザ対応**: Chrome/Firefox/Safari の最新版で動作すれば十分
- **オフライン動作**: ネットワーク接続不要で動作する

### デザイン基準

以下の原則を適用する（frontend-design の基準に準拠）:

- **タイポグラフィ**: system-ui フォントスタック。サイズ階層を明確に
- **余白**: 8px グリッド。詰め込まない
- **配色**: ダークモード対応を検討。コントラスト比 AA 準拠
- **レスポンシブ**: モバイルでも操作可能（最低限のレスポンシブ対応）
- **インタラクション**: hover/focus の視覚フィードバック。transition は 150〜300ms

### コード品質

- セマンティックな HTML 構造
- CSS カスタムプロパティ（CSS変数）でテーマ管理
- JavaScript は `DOMContentLoaded` 後に実行
- グローバル変数の汚染を避ける（IIFE またはモジュールパターン）
- エラーハンドリングを入れる

## 用途例

指定がない場合や参考として:

- データビジュアライザー（チャート、グラフ）
- カラーパレットエクスプローラー / ジェネレーター
- レイアウトグリッドツール
- CSS プロパティのプレイグラウンド
- マークダウンエディタ / プレビューア
- コンセプトマップ / マインドマップ
- インタラクティブなチュートリアル / デモ
- ゲーム / パズル

## 実行プロセス

### Step 1: 設計

- ツールの目的と主要機能を定義
- UI のワイヤーフレーム的な構成を決定
- 必要なインタラクションを列挙

### Step 2: 実装

1. HTML 構造を作成
2. CSS でスタイリング（カスタムプロパティから定義）
3. JavaScript でインタラクションを実装
4. エッジケースの処理

### Step 3: 検証と公開

1. ファイルを保存
2. `open` コマンドでブラウザで自動的に開く:

```bash
open /path/to/playground.html
```

3. 実際にブラウザで動作することを確認
4. ユーザーにファイルパスを伝える
